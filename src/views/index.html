Production-Ready Secure Web Template

Express + Helmet, strict nonce-based CSP, HSTS preload, Trusted Types, and a deny-by-default Permissions-Policy — all aligned with the OWASP Secure Headers Project.

Run live header audit Read the threat model

Content-Security-Policy

Strict nonce + strict-dynamic policy with object-src 'none', base-uri 'self', frame-ancestors 'none', and Trusted Types.

Strict-Transport-Security

2-year max-age with includeSubDomains and preload. RFC 6797 compliant.

X-Frame-Options + frame-ancestors

Both emitted in lock-step to mitigate clickjacking on modern browsers (CSP) and legacy IE11 (XFO).

X-Content-Type-Options

nosniff. Blocks MIME confusion across user-upload and CDN responses.

Referrer-Policy

strict-origin-when-cross-origin. Prevents cross-origin path/query leakage and HTTPS→HTTP downgrade leakage.

Permissions-Policy

Deny-by-default for camera, microphone, geolocation, payment, usb, accelerometer, gyroscope, autoplay, and interest-cohort.
