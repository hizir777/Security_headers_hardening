About this project

This template is the practical companion to the Security Headers Hardening Guide & Audit Report, prepared for the Secure Web Development course at Istinye University under instructor Keyvan Arasteh. It demonstrates a production-grade Express server that emits the six core defensive HTTP response headers with the “golden standard” values documented in the guide and verifies them via a live audit endpoint.

Why these headers?

Browser-enforced response headers are the cheapest, highest-leverage defense-in-depth layer a web application can deploy. They neutralise whole classes of attack — clickjacking, MIME confusion, SSL-stripping, referrer leakage, content injection — without requiring any change to application logic. They do not fix server-side injection or broken authentication, but they reliably contain the blast radius when something upstream goes wrong.

Architecture at a glance

  • Single source of truth for every header value in src/config/headers.js.
  • Per-request nonce middleware attaches a fresh 128-bit value to res.locals.cspNonce.
  • Helmet v7 consumes the nonce via a directive function so CSP is always in sync with the rendered HTML.
  • Standalone audit endpoint at /readyz and the human-facing /audit page programmatically verify the live response.
  • CSP violation receiver at /csp-report accepts both legacy report-uri and Reporting API payloads.
